# OCAX Marketing Reset v1

**Version**: 1.0  
**Purpose**: New Marketing Strategy for Current Phase  
**Last Updated**: 2026-03-18

---

## Why Current Marketing Should Change

The current OCAX project is in **early awareness stage**, not FAQ-driven stage.

Current problems:
- Too much internal documentation
- Focus on technical details instead of understanding
- No clear bilingual strategy
- No channel layering

---

## Main Messaging Problem

### Current Issues

| Problem | Impact |
|---------|--------|
| Too many technical terms | Hard to understand |
| No simple core message | People don't get it |
| English-only | Chinese audience missed |
| No clear differentiation | Just another "AI agent" project |

---

## Jargon Problem

### Terms That Confuse Outsiders

| Term | Problem |
|------|---------|
| Validator | What is it? |
| Settlement | Too financial |
| Passport | Sounds like travel document |
| TP/CR | What do they mean? |
| Workflow | Too technical |

### Simpler Alternatives

| Instead of... | Say... |
|---------------|--------|
| Validator | "Verification System" |
| Settlement | "Credit Release" |
| Passport | "Reputation Card" |
| TP | "Trust Points" |
| CR | "Credits" |
| Workflow | "Task Type" |

## What to Say First

### The Core Message (Always First)

> **Execution ≠ Completion. Validation = Completion.**

### Secondary Messages

1. "Node ability is visible"
2. "Credits are earned, not given"
3. "Task completed only after verification"

- Protocol details

- Internal processes

## Top 5 Message Changes to Make Immediately

### 1. Lead with Problem, Not Solution

**Before:** "OCAX has a Validator layer that verifies task execution"

**After:** "How do you know an AI agent actually completed the task? OCAX verifies."

### 2. Replace Technical Terms

**Before:** "Validator + Settlement + Passport"

**After:** "Verification + Credit Release + Reputation Tracking"

### 3. Simplify the Core Message

**Before:** Long technical explanation

**After:** "OCAX: Task verification for AI agents. Execution ≠ Completion."

### 4. Add Social Proof

**Before:** "Our protocol is designed to..."

**After:** "We've verified 2 real tasks. 100% acceptance rate."

### 5. Use Bilingual

**Add Chinese version of:**
- README
- Core messages
- Launch post

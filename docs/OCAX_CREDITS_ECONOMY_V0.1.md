## 1. 设计理念

OCAX 前期采用**积分制经济**，后期可平滑升级到**真实交易**。

```
[积分系统] ----(条件满足)----> [真实交易]
    v0.1                          v1.0
```

---

## 2. 积分体系架构

### 2.1 两层积分模型

| 积分类型 | 名称 | 用途 | 可交易 |
|----------|------|------|--------|
| **声誉积分** | Trust Points (TP) | 等级、权重、声誉 | ❌ 不可 |
| **经济积分** | Credits (CR) | 结算、支付、任务 | ✅ 可 (后期) |

### 2.2 积分关系

```
Trust Points (TP) ──┐
                    ├──> Node Rank ──> 优先匹配
Credits (CR) ───────┘
                    └──> 任务结算 <──> 未来真钱
```

## 5. 积分与未来真钱的关系

### 5.1 升级路径

```
Phase 1: 纯积分系统 (v0.1-v0.4)
├── 仅 CR 流通
├── 无真实资金
└── 熟悉市场机制

Phase 2: 测试兑换 (v0.5)
├── 1 CR = $0.01 固定
├── 小额测试
└── 收集反馈

Phase 3: 市场定价 (v1.0)
├── 开放兑换
├── 真实资金流入
└── 保持双轨并行
```

### 5.2 升级条件

从积分系统升级到真实交易需满足：

| 条件 | 要求 |
|------|------|
| 运营时间 | ≥ 6 个月 |
| 节点数 | ≥ 100 |
| 交易量 | ≥ 10,000 订单 |
| 验收通过率 | ≥ 95% |
| 无重大安全事件 | 通过审计 |

### 5.3 映射原则

- **保留积分**: 已积累 CR 可 1:1 转换为 USDC
- **等级继承**: TP 等级对应 VIP 权益
- **平滑过渡**: 双轨并行 3 个月后退出积分

## 9. 问答

### Q1: 积分可以变现吗?
v0.1 阶段: ❌ 不可变现  
v1.0 阶段: ✅ 可 1:1 转换为 USDC

### Q2: 积分会过期吗?
- TP: ❌ 永不过期
- CR: ❌ 永不过期

### Q3: 如何防止刷分?
- 任务真实性验证
- IP/设备指纹
- 行为模式检测
- 人工审核抽检

### Q4: 这套机制是否替代真实支付?
**是**，v0.1-v0.4 期间完全替代  
**可升级**，满足条件后平滑过渡到真钱

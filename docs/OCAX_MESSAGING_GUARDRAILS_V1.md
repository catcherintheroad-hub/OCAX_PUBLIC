## 1. OCAX's Real Direction

### What OCAX Actually Is

**A peer-to-peer marketplace for computing power.**

| Side | What They Get |
|------|--------------|
| Device owners | Earn money from idle devices |
| Task requesters | Access more compute power |

### Core Value Proposition

- **For device owners:** Turn idle devices into income
- **For task requesters:** Access more compute without buying hardware

### How It Works

1. Device owners connect idle devices to OCAX
2. Task requesters post tasks that need compute
3. Devices run the tasks
4. Task requesters pay on completion
5. Device owners earn for completed work

## 7. Cold Start Account Principles

### For Zero-Follower Accounts

| Principle | Why |
|-----------|-----|
| Start with a question | Grabs attention |
| Keep posts under 200 chars | Easy to read |
| Don't sound authoritative | No credibility yet |
| Be honest about early stage | Builds trust |
| Don't hype | Unearned hype = skepticism |
| Focus on problem-solving | Real value first |
| Engage with others first | Build relationships |
| Don't over-post | 1-2 quality posts > 10 spam |

### What Works on Cold Start

- Questions people relate to
- Honest about being early
- Real problems, real solutions
- Personal tone
- Short, memorable lines

### What Kills Cold Start

- Big promises
- Technical jargon
- "We're going to change everything"
- Ignoring skeptics
- Posting too much

## 8. How to Avoid Sounding Like a Scam

### Red Flags to Avoid

| ❌ Scam Signal | ❌ Scam Signal |
|----------------|----------------|
| "Guaranteed income" | "Limited time offer" |
| "Passive income" alone | "No work required" |
| "Earn while you sleep" | "Sign up now" |
| Celebrity endorsements | "Be your own boss" |
| "AI magic" | "Revolutionary platform" |

### Trust Builders

| ✅ Build Trust | How |
|----------------|-----|
| Be honest about stage | "Early stage, building in public" |
| Show real work | "First devices connected" |
| Acknowledge concerns | "We know it sounds like a scam" |
| Explain the difference | "Real tasks, not mining" |
| Don't overpromise | "Earn for actual work" |
| Be specific | "Tasks vary by request" |

### Key Differentiators from Scams

| Fake Projects | OCAX |
|---------------|------|
| Fake tasks | Real tasks from real users |
| Empty promises | Actual work required |
| No transparency | Open source, public repo |
| "Keep running" | "Run real tasks" |
| No product | Working OpenClaw integration |

## 9. Twenty Rules for All Future Writing

### Rule 1: Lead with Value, Not Technology
❌ Don't: "We built a verification layer."
✅ Do: "Your idle device runs real tasks. You earn money."

### Rule 2: Speak to Regular People First
❌ Don't: "For AI agents and automation workflows..."
✅ Do: "Got a spare phone? Let it work for others."

### Rule 3: Never Lead with "AI Says Done"
❌ Don't: "AI says 'I'm done.' But did it really finish?"
✅ Do: "Your idle device runs real tasks. You earn money."

### Rule 4: Keep It Short
❌ Don't: Long paragraphs explaining architecture
✅ Do: One clear sentence + action

### Rule 5: Avoid Technical Jargon in Public Facing
❌ Don't: "Verification layer," "execution proof," "completion signal"
✅ Do: "Check if done," "real work," "get paid"

### Rule 6: Don't Overpromise
❌ Don't: "Earn hundreds a day!"
✅ Do: "Earn money for completed tasks"

### Rule 7: Acknowledge Skepticism
❌ Don't: Ignore scam concerns
✅ Do: Address directly: "Real tasks, not mining"

### Rule 8: Use Real Words
❌ Don't: "Leverage," "optimize," "synergy"
✅ Do: "Use," "work," "earn"

### Rule 9: Don't Sound Like a Whitepaper
❌ Don't: Abstract technical descriptions
✅ Do: Concrete, specific explanations

### Rule 10: Focus on Two Sides
❌ Don't: Only talk about "verification"
✅ Do: Explain both device owners AND task requesters

### Rule 11: Be Honest About Stage
❌ Don't: Imply a mature product
✅ Do: "Early stage, building in public"

### Rule 12: Use Active Voice
❌ Don't: "Computation is provided by..."
✅ Do: "Your device runs tasks. You earn money."

### Rule 13: Don't Use Crypto/Node Language
❌ Don't: "Nodes earn rewards," "validators"
✅ Do: "Devices run tasks," "owners earn"

### Rule 14: Keep Bios Under 160 Characters
Twitter bio limit. Keep it tight.

### Rule 15: Don't Make Absolute Claims
❌ Don't: "Always," "Guaranteed," "Never"
✅ Do: "Usually," "Often," "In most cases"

### Rule 16: Use Questions to Hook
❌ Don't: "OCAX is a marketplace for..."
✅ Do: "Got a spare device? Let it work."

### Rule 17: Separate Technical and Public Docs
Technical details belong in /docs/technical/
Public messaging belongs everywhere else.

### Rule 18: Don't Mix Messaging Systems
Pick one primary message per post. Don't layer too many concepts.

### Rule 19: Test on Non-Technical Friend
If a non-technical friend can't understand it in 10 seconds, rewrite.

### Rule 20: When in Doubt, Ask
Before publishing major content, ask: "Does this sound like a real project or a scam?"

# OCAX

> 调用真实电脑执行 OpenClaw 任务。

OCAX 是一个调用真实电脑执行 OpenClaw 任务的可验证执行网络。

开发者提交任务，节点提供安装了 OpenClaw 的真实电脑，平台负责验收结果并记录结算。

## OCAX 是什么

OCAX 不是普通任务平台。
它不是“共享电脑赚钱”平台。
它也不是一个面向大众开放的闲置设备算力市场。

OCAX 是一个受控的执行网络，适合这类任务：

- 需要真实电脑环境
- 需要真实浏览器环境
- 需要真实桌面上下文
- 需要可验证执行结果

## 为什么不是 Cloud / VPS

很多 OpenClaw 或 browser automation 任务，并不适合只跑在通用云主机或无头 VPS 里。

OCAX 关注的是：

- 本地机器上下文
- 真实浏览器状态
- 真实桌面交互环境
- 执行后可验收、可记录

所以它更准确的描述不是“算力市场”，而是“可验证执行网络”。

## 为什么它不是“共享电脑赚钱平台”

这种说法过于宽泛，也已经不再代表当前产品方向。

OCAX 现在并不以这些作为主叙事：

- 闲置设备变现
- passive income
- 面向大众的开放市场

当前更核心的是：

- 任务匹配度
- 节点质量
- 验收能力
- 结算纪律
- 网络稳步开放

## 它适合谁

OCAX 当前更适合：

- OpenClaw 高级用户
- 自动化开发者
- 需要真实电脑执行环境的团队
- 能提供稳定执行环境的受筛选节点

## 当前阶段：受控测试

OCAX 当前处于邀请码 / 受控测试阶段。

现阶段优先保证：

- 执行可靠性
- 节点质量
- 验收质量
- 结算稳定性
- 网络安全开放节奏

## 核心执行流程

```text
调用者提交任务
-> 节点电脑使用 OpenClaw 执行
-> 平台验收输出结果
-> 更新结算与记录
```

## 这个公开仓是什么

这个仓库是 OCAX 的公开 companion repo。

它主要用于：

- 对外解释产品
- 存放可公开文档
- 分享 schema
- 展示 demo 材料
- 存放对外传播素材

它不是完整应用主仓，也不是生产环境的唯一事实来源。

## 仓库包含什么

```text
/docs         对外文档
/schemas      公开协议与数据结构
/demo_inputs  示例输入
/demo_outputs 示例输出
/release      对外发布与传播素材
```
